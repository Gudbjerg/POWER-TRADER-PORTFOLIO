## P0 — Correctness Bugs

### 1. BESS `_npv_only` dead code / wrong FCR sensitivity
**File:** `pages/7_BESS_Optimiser.py`  
**Commit:** pending  
**Fix:** Removed dead line `ann_adj = sum(row["disc_cf"] * 0 for row in s["rows"])`. Rewrote FCR sensitivity to use a clean discounted annuity:
```python
fcr_adj = (s["fcr_n_y1"] + s["fcr_d_y1"]) * (fcr_mult - 1.0)
fcr_npv_adj = sum(fcr_adj / (1.0 + disc) ** yr for yr in range(1, lifetime + 1))
return s["npv"] + fcr_npv_adj
```
Previously `_npv_only` for FCR sensitivity returned an incorrect NPV (multiplied a dead zero sum).

### 2. Clean Spark Spread (Signal 5) used snapshot EUA across all historical observations
**File:** `pages/5_Mispricing_Dashboard.py`  
**Commit:** pending  
**Fix:** Replaced `fetch_eua_price()` (today's spot) with a 3-year historical `yfinance` CO2.L series, merged by date to the feature matrix. Signal is now dropped entirely if the historical series is unavailable (no silent fallback). The percentile distribution is now computed over historically correct EUA prices, not today's price applied uniformly backwards.

### 3. D4 Risk Dashboard: silent signal degradation with no user visibility
**File:** `pages/6_Risk_Dashboard.py`  
**Commit:** `49dae5d`  
**Fix:** Added `_unavailable` list populated when signals cannot be computed. If any signals are missing, a `st.warning` banner shows the count, names, and which API keys to check. Previously the sidebar signals would silently produce flat returns with no explanation.
